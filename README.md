# Qubit Simulation Console (量子電路模擬器)
 
這是一個以 C++ 開發的高效能量子電路模擬器。本專案透過一個互動式的命令列控制台（QConsole），讓使用者可以動態輸入、解析並執行量子電路。模擬器底層支援量子疊加、量子糾纏的狀態追蹤，並包含了自定義的複數與定點數（Fixed-Point）運算以確保機率幅計算的精度。
 
## ✨ 核心特色 (Features)
 
* **互動式控制台 (Interactive CLI)**：提供友善的操作介面，即時解析並執行量子電路字串。
* **豐富的量子閘支援**：內建包含 Pauli 閘、參數化旋轉閘、CNOT/SWAP 等雙位元閘，以及 Toffoli 等三位元閘。
* **動態電路與控制流**：支援測量（Measurement）、宇稱檢查（Parity）、全量子位元重置（Reset），以及基於動態測量、宇稱結果或經典狀態讀取的條件執行（If）和標籤跳轉（Goto）。If 指令完整支援**嵌套**與**多子指令**。

## 🚀 快速開始 (Quick Start)
 
### 下載執行檔 (免編譯)
1. 前往本專案的 Releases 頁面。
2. 下載最新的 `Qubit_Simulation.exe`。
3. 雙擊執行即可進入 QConsole。
### 從原始碼編譯 (Build from Source)
本專案使用標準 C++ 開發，推薦使用 **Visual Studio 2022** 進行編譯：
1. 複製此儲存庫：`git clone <your-repo-url>`
2. 使用 Visual Studio 2022 開啟專案資料夾。
3. 將建置組態切換為 **Release** 與 **x64**。
4. 為了確保執行檔的獨立性，建議在專案屬性中將 C/C++ -> 程式碼產生 -> 執行階段程式庫設為 `多執行緒 (/MT)`。
5. 點擊「建置方案」，生成的 `.exe` 將位於 `x64/Release/` 目錄下。
## 📖 控制台指令說明
 
進入 QConsole 後，你可以使用以下基本指令：
 
* `help`：顯示基本指令列表。
* `manual`：顯示完整的量子電路操作手冊與支援的元件清單。
* `config`：查看目前設定。
* `run "<expr>"`：執行量子電路（**注意：表達式必須包含在雙引號中**）。
* `show`：顯示最後一次執行的電路圖與狀態快照。
* `exit` / `quit`：離開程式。
## 🛠️ 電路語法與量子閘列表
 
電路表達式的基本語法為：`"<總量子位元數>, {量子閘, 目標1, ...}, {量子閘...}"`
 
> ⚠️ **大小寫嚴格區分**：所有閘名稱與指令必須完全符合文件中的大小寫。例如 `{If,...}` 正確，`{IF,...}` 或 `{if,...}` 均無效。
 
### 支援的量子組件：
 
**1. 單量子位元閘**
* `{H,0}`: Hadamard 閘
* `{X,0}`, `{Y,0}`, `{Z,0}`: Pauli 閘
* `{S,0}`, `{Sdg,0}`, `{T,0}`, `{Tdg,0}`: 相位閘
* `{Rx,0,90}`, `{Ry,0,90}`, `{Rz,0,90}`: 參數化旋轉閘（第三個參數為度數）
**2. 雙量子位元閘**
* `{CNOT,0,1}`: 控制反轉閘（控制位元, 目標位元）
* `{CZ,0,1}`: 控制相位閘
* `{SWAP,0,1}`, `{iSWAP,0,1}`, `{SqrtSWAP,0,1}`: 交換系列閘
**3. 三量子位元閘**
* `{CCNOT,0,1,2}`: Toffoli 閘（控1, 控2, 目標）
* `{CSWAP,0,1,2}`: Fredkin 閘
* `{Deutsch,0,1,2,45}`: Deutsch 閘（含角度參數）
**4. 測量與控制流**
 
* `{M,0}` 或 `{Measure,0}`：測量特定位元，測量後退出糾纏。
* `{Parity,0,1,2}`：測量多位元的宇稱，回傳 0（偶）或 1（奇）。位元**不**會被標記為坍縮，仍可繼續被閘操作。
* `{Reset}` 或 `{RST}`：將**全部**量子位元重置回 $|0\rangle$ 初始狀態，清除所有糾纏與測量紀錄。不接受任何參數。常用於在 If 分支內確保量子位元回到已知狀態後再繼續操作。
* `{Label,L1}`, `{Goto,L1}`：標籤與無條件跳轉。Goto 的目標為**標籤名稱字串**，非行號。
* `{If, {條件}, {指令1}, {指令2}, ...}`：條件執行分支。條件成立時依序執行所有子指令。子指令可為任意普通指令，也可再嵌套 If。
  **條件區塊支援三種寫法：**
  | 條件寫法 | 說明 |
  |---|---|
  | `{M, q}` | 當場測量 Q*q*，預期結果為 1（預設） |
  | `{M, q, val}` | 當場測量 Q*q*，預期結果為 *val*（0 或 1） |
  | `{q}` 或 `{q, val}` | 讀取已坍縮的 Q*q* 的結果，Q*q* 必須已被事先測量 |
  | `{Parity, q0, q1, ...}` | 當場測量多位元宇稱，預期結果為 1（奇宇稱，預設） |
  | `{Parity, q0, q1, ..., val}` | 當場測量多位元宇稱，預期結果為 *val*（0 或 1）。val 僅在位元數 ≥ 2 且末尾數字為 0 或 1 時視為預期值 |
  省略 `val` 時一律預設為 **1**。
## 💡 經典示範 (Examples)
 
**1. 建立貝爾態 (Bell State)**
 
準備兩個量子位元，將其變為 $|00\rangle$ 與 $|11\rangle$ 各佔 50% 機率的最大糾纏態：
 
```text
QConsole> run "2, {H,0}, {CNOT,0,1}"
QConsole> show
```
 
**2. 量子遙傳 (Quantum Teleportation)**
 
```text
QConsole> run "3, {H,1}, {CNOT,1,2}, {CNOT,0,1}, {H,0}, {M,0}, {M,1}, {If,{1,1},{X,2}}, {If,{0,1},{Z,2}}"
```
 
**3. 條件重置後繼續操作**
 
若 Q0 測量結果為 1，則重置所有量子位元回 $|0\rangle$ 再繼續：
 
```text
QConsole> run "3, {H,0}, {CNOT,0,1}, {If, {M,0}, {Reset}}, {H,2}"
```
 
**4. 宇稱作為條件（搭配 Label/Goto 重試）**
 
對 Q0、Q1 測量宇稱，若為奇宇稱（1）則重置並跳回起點重試：
 
```text
QConsole> run "3, {Label,start}, {H,0}, {CNOT,0,1}, {If, {Parity,0,1}, {Reset}, {Goto,start}}, {H,2}"
```
 
**5. 嵌套 If**
 
Q0 測量為 1，且 Q1 測量也為 1，才對 Q2 做 X 閘：
 
```text
QConsole> run "3, {H,0}, {H,1}, {M,0}, {M,1}, {If, {0}, {If, {1}, {X,2}}}"
```
 
## ⚙️ 常見問題
 
**Q：`{Goto,0}` 沒有跳轉效果？**
A：Goto 的參數是**標籤名稱**，不是行號。請先用 `{Label,名稱}` 宣告標籤，再用 `{Goto,名稱}` 跳轉。
 
**Q：`{IF,...}` 被忽略了？**
A：指令名稱大小寫嚴格區分，正確寫法是 `{If,...}`（首字母大寫，其餘小寫）。
 
**Q：Parity 測量後的位元還能繼續操作嗎？**
A：可以。Parity 測量不會讓位元坍縮到確定態，只會清除相應機率幅並重新正規化，位元仍可繼續被閘操作。這與 `{M,q}` 的行為不同。
